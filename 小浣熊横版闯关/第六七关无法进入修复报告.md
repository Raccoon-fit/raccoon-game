## 问题分析

### 问题根因
通过深入分析游戏代码，发现问题出现在**第七关的钥匙收集逻辑缺失**：

1. **第七关设计**：
   - 需要收集钥匙（key）来开门
   - 门（door）需要变为"打开"状态（collected: true）才能完成关卡
   - 只有在钥匙收集后，门才会自动变为collected状态

2. **逻辑缺陷**：
   - 第七关定义了钥匙和门作为特殊物品
   - 有门的检查逻辑：`doorItem?.collected`
   - 有门的打开逻辑：`doorItem.collected = true`
   - **但是缺少钥匙的收集逻辑**！

3. **影响**：
   - 玩家无法收集钥匙
   - 门无法变为打开状态
   - 无法完成第七关
   - 由于关卡顺序锁定，第六关也无法重新进入

## 修复内容

### 核心修复
在 `checkCollisions()` 函数中添加了特殊物品收集逻辑：

```javascript
// 特殊物品收集
this.specialItems?.forEach(item => {
    if (!item.collected) {
        const itemSize = item.type === 'key' ? 30 : (item.type === 'door' ? 30 : 20);
        if (this.player.x < item.x + itemSize &&
            this.player.x + this.player.width > item.x &&
            this.player.y < item.y + itemSize &&
            this.player.y + this.player.height > item.y) {
            
            item.collected = true;
            if (item.type === 'key') {
                console.log('🔑 收集钥匙!');
            } else if (item.type === 'clue') {
                console.log(`🧩 收集线索: ${item.text}`);
            }
        }
    }
});
```

### 修复逻辑说明
1. **碰撞检测**：检测玩家与特殊物品的碰撞
2. **钥匙收集**：当玩家碰到钥匙时，设置 `key.collected = true`
3. **自动开门**：在 `updateLevelSpecific()` 中检测到钥匙收集后，自动将门设置为 `door.collected = true`
4. **关卡完成**：在 `reachGoal()` 中检查门状态，允许完成关卡

### 增强功能
- 统一了所有特殊物品（钥匙、线索等）的收集逻辑
- 添加了收集反馈日志
